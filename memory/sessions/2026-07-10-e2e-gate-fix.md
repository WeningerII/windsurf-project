**Decisions and why:**
- Added a first-run empty state to App.tsx (heading "No characters yet" +
  "Create Your First Character" CTA) rather than only re-anchoring specs: a
  blank landing was a real Phase-1 UX regression, and clear-all tests require
  "Your Characters" to disappear when empty, so the two states need distinct
  copy anyway.
- outcome-baseline: rewrote resetToLanding twice before landing on fresh
  browser context per system. In-page clears lose two races: (1) pagehide
  save-flush can run concurrently with the next document's load in Chrome
  (beats even addInitScript-at-document-start); (2) documents are mirrored to
  IndexedDB (documentStorage.ts) and a fresh boot restores from it. Fresh
  context isolates both stores and matches the test's "fresh user" premise.
- playwright maxFailures:10 in CI so systemic breakage fails fast WITH output;
  pages.yml now `workflow_run` on CI/CD Pipeline, `if: conclusion == success`,
  checks out the verified head_sha; workflow_dispatch kept as escape hatch.
